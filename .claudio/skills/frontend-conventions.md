# Frontend Conventions — hada-commerce

Stack: React 19 + TypeScript 6 + Vite 8 + TanStack Router + TanStack Query + Tailwind v4 + Radix UI Themes + Recharts + Lucide React

## File structure
```
frontend/src/
├── routes/              ← TanStack Router file-based routes
│   └── store/           ← storefront-facing pages
├── components/          ← shared UI components
│   └── cart.tsx
├── lib/
│   ├── api.ts           ← ALL API fetch functions live here
│   └── hooks.ts         ← shared TanStack Query hooks
├── types/               ← TypeScript types matching backend entities
├── main.tsx             ← React entry point
└── routeTree.tsx        ← auto-generated by TanStack Router (do not edit)
```

## Quick rules
- All API calls go through `frontend/src/lib/api.ts` — never inline `fetch()` in components
- Send `X-Tenant-ID` header on every API request
- Always type API responses — define types in `frontend/src/types/`
- Use Radix UI Themes primitives for interactive elements, Tailwind for layout
- Use TanStack Query for all server state — no useState for remote data
- No `any` types unless absolutely unavoidable (cast via `unknown` instead)

## TanStack Router — file-based routes
```tsx
// frontend/src/routes/products.tsx
```


## API fetch pattern
```ts
// frontend/src/lib/api.ts
const BASE_URL = 'http://localhost:8080/api/v1'
const DEFAULT_TENANT = 'tenant-1'  // TODO: from auth context

async function apiFetch<T>(
  path: string,
  init: RequestInit = {},
  tenantId = DEFAULT_TENANT,
): Promise<T> {
  const res = await fetch(BASE_URL + path, {
    ...init,
    headers: {
      'Content-Type': 'application/json',
      'X-Tenant-ID': tenantId,
      ...init.headers,
    },
  })
  if (!res.ok) {
    const err = await res.json().catch(() => ({ message: res.statusText }))
    throw new Error(err.message ?? `HTTP ${res.status}`)
  }
  return res.json() as Promise<T>
}

// Read
export const api = {
  listProducts: (params: { page?: number; pageSize?: number }) =>
    apiFetch<PaginatedResult<Product>>(`/products/?page=${params.page ?? 1}&page_size=${params.pageSize ?? 20}`),

  getProduct: (id: string) =>
    apiFetch<Product>(`/products/${id}`),

  createProduct: (body: CreateProductRequest) =>
    apiFetch<Product>('/products/', { method: 'POST', body: JSON.stringify(body) }),

  updateProduct: (id: string, body: UpdateProductRequest) =>
    apiFetch<Product>(`/products/${id}`, { method: 'PUT', body: JSON.stringify(body) }),

  deleteProduct: (id: string) =>
    apiFetch<void>(`/products/${id}`, { method: 'DELETE' }),
}
```

## TypeScript types
Match backend JSON exactly. Backend returns camelCase from Go's `json` tags.

```ts
// frontend/src/types/index.ts

export interface Money {
  amount: number  // cents
  currency: string
}

export interface PaginatedResult<T> {
  items: T[]
  total: number
  page: number
  page_size: number
  total_pages: number
}

export interface Product {
  id: string
  tenant_id: string
  name: string
  description: string
  price: Money
  sku: string
  stock: number
  status: 'draft' | 'active' | 'archived'
  tags: string[]
  created_at: string
  updated_at: string
}
```

## TanStack Query patterns

### Read (useQuery)
```tsx
import { useQuery } from '@tanstack/react-query'
import { api } from '@/lib/api'

function ProductsList() {
  const { data, isPending, error } = useQuery({
    queryKey: ['products', page],
    queryFn: () => api.listProducts({ page }),
  })

  if (isPending) return <Spinner />
  if (error) return <Text color="red">{error.message}</Text>

  return (
    <Table.Root>
      {data?.items.map(p => <ProductRow key={p.id} product={p} />)}
    </Table.Root>
  )
}
```

### Write (useMutation)
```tsx
import { useMutation, useQueryClient } from '@tanstack/react-query'

function CreateProductForm() {
  const queryClient = useQueryClient()
  const mutation = useMutation({
    mutationFn: api.createProduct,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['products'] })
    },
  })

  const handleSubmit = (data: CreateProductRequest) => {
    mutation.mutate(data)
  }

  return (
    <form onSubmit={...}>
      <Button disabled={mutation.isPending}>
        {mutation.isPending ? 'Creating...' : 'Create'}
      </Button>
      {mutation.error && <Text color="red">{mutation.error.message}</Text>}
    </form>
  )
}
```


## Gotchas
- Backend returns `snake_case` JSON keys — check the Go struct tags before typing
- `PaginatedResult.items` can be `null` from the backend if empty — always use `data?.items ?? []`
- Vite path alias: check `vite.config.ts` for `@/` alias (points to `src/`)
- TanStack Router's `routeTree.tsx` is auto-generated — never edit it manually
- Radix UI's `<Theme>` must wrap the app at the root — it's already in App.tsx
- Tailwind v4 uses CSS custom properties (`--color-*`) not the v3 config approach
